Agent Interaction Model

How OpenClaw, Aeon skills, and other agents communicate with the desktop app autonomously

A

REST-only API

Tauri app runs a local HTTP server (:5558). Agents POST commands. Simple, works from GitHub Actions runners too.

POST localhost:5558/notify
POST localhost:5558/dispatch
POST localhost:5558/ui/navigate
GET localhost:5558/state

Pros

  • Works from GH Actions (remote)
  • Simple to implement

Cons

  • No real-time push from app to agent
  • Polling needed for state updates
B

WebSocket Server ⭐ Recommended

Tauri app runs a WS server (:5558). OpenClaw and local agents maintain a persistent connection. Bidirectional real-time — app can push to agents too.

ws://localhost:5558

→ {"type":"notify","msg":"hl-intel done"}
→ {"type":"navigate","panel":"AGN"}
→ {"type":"update","panel":"MCP","data":...}
← {"type":"state","panels":...}

Pros

  • Real-time both ways
  • App can instruct agents
  • OpenClaw connects naturally (already WS)

Cons

  • GH Actions needs REST bridge

Recommended: B + REST fallback

WS server for OpenClaw and local agents (persistent, real-time). REST endpoint for GitHub Actions runners (one-shot from remote). Same port, same auth — WS upgrade if supported, plain HTTP otherwise.

What agents can do autonomously

📢 Push notifications
Skill completes → toast appears in app. hl-intel done → summary card in terminal panel.
🧭 Navigate UI
Agent says "show me the MCP panel" → app switches to MCP inspector. OpenClaw directs the UI.
📊 Update panels live
Agent pushes data → panel re-renders. hl-intel pushes trade setups directly into the intel view.
⚡ Trigger actions
OpenClaw receives a user message → dispatches skill → posts job update back to app.
🔍 Query app state
Agent asks "which MCP servers are healthy?" → app responds with live state. No separate monitoring needed.
🚨 Surface alerts
MCP server goes down → OpenClaw detects → pushes alert to app → red indicator + banner. You see it instantly.
Agent SDK built into the app
The app ships with a tiny JS client (nerv-sdk.js) that agents can import. One line to connect: const nerv = new NervClient('ws://localhost:5558', token). Then: nerv.notify(), nerv.navigate(), nerv.push().
