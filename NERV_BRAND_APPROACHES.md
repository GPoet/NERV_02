# NERV Brand Identity: Three Strategic Approaches

**Prepared by:** Brand Guardian
**Date:** 2026-03-26
**Subject:** NERV — Umbrella brand for autonomous AI systems
**Sub-entities:** Aeon, Paperclip, OpenClaw, NERV Dashboard

---

## Current State Audit

Before proposing directions, a snapshot of what exists:

- **Primary color:** `#ff6600` (pure orange, used across all UI elements)
- **Background:** `zinc-950` / `#04040a` range (near-black)
- **Typography:** Geist Sans + Geist Mono (Vercel system fonts)
- **UI patterns:** Pulse-ring animations, `rounded-none` (sharp corners), monospace tracking, bracket indicators, `10-11px` micro-text, triangular icon mark
- **Current name treatment:** `NERV_02` with underscore and version number
- **Existing logo:** Simple triangle (`<polygon points="16,3 30,27 2,27" fill="#ff6600"/>`)

The foundation is strong. The sharp corners, micro-typography, and pulse animations already carry institutional weight. What's missing is a deliberate system — the orange is undifferentiated, the typography lacks hierarchy, and the brand has no formal voice or protection framework.

---

## APPROACH A: "INSTITUTIONAL WEIGHT"

**Maximum authority, minimal warmth. The brand speaks in declarations, not conversations.**

### 1. Name Treatment and Motto

**Name:** NERV (no underscore, no version number — the name is absolute)

**Formal designation:** NERV SYSTEMS

**Motto:** "Systems Nominal."

The motto functions as a status report. It is not aspirational — it is operational. It implies that NERV's natural state is controlled, monitored, correct. The gravity comes from what it does NOT say: what happens when systems are not nominal. Used as a sign-off on reports, a footer element, a loading state.

**Sub-entity naming convention:**
- NERV/AEON (autonomous operations division)
- NERV/PAPERCLIP (orchestration framework)
- NERV/OPENCLAW (gateway services)
- NERV/DASHBOARD (command interface)

### 2. Color Palette

```
PRIMARY
  Institutional Orange    #E05A00    (deepened from #ff6600 — less screen-glow, more authority)

SECONDARY
  Void Black              #06070D    (the abyss behind the interface)
  Graphite                #1A1C23    (surface/card backgrounds)
  Steel                   #2A2D36    (elevated surfaces, borders)

ACCENT
  Amber Warning           #F0A500    (warm highlight — used sparingly for emphasis)

NEUTRALS
  Ash                     #6B7080    (disabled text, secondary labels)
  Pale Steel              #A8B4C4    (body text)
  Bone                    #E8ECF0    (primary text, headings)

STATUS COLORS
  Operational Green       #22C55E    (systems nominal)
  Caution Amber           #F59E0B    (warning state)
  Critical Red            #EF4444    (failure/alert)
  Processing Blue         #3B82F6    (in-progress/loading)
```

**Rationale:** The primary orange is pulled down from `#ff6600` to `#E05A00` — losing the "screen brightness" quality and gaining a burnt, institutional gravity. It reads less like a tech startup accent and more like a warning label on heavy machinery. The amber accent creates a secondary warm tone for hierarchy without competing with the primary.

### 3. Typography System

**Primary (Headings):** Inter Tight — Condensed, mechanical, high x-height. Carries authority without decorative weight. Set in ALL CAPS with `letter-spacing: 0.15em` for institutional headers.

**Secondary (Body):** Geist Sans (retained) — Clean, contemporary, pairs well with condensed headings. Already embedded in the codebase.

**Tertiary (Data/Code):** Geist Mono (retained) — For terminal output, data displays, status indicators. The workhorse of the UI.

**Display (Rare):** Bebas Neue — Used only for the NERV wordmark and large-scale environmental applications. Ultra-condensed, mechanical compression. Evokes the typographic density of institutional signage.

**Hierarchy rules:**
- Headers: Inter Tight, 10-12px, ALL CAPS, tracking 0.15-0.25em
- Body: Geist Sans, 13-14px, sentence case, tracking normal
- Data: Geist Mono, 10-11px, uppercase for labels, normal for values
- Display: Bebas Neue, 48px+, used only in hero/splash contexts

### 4. Logo Concept

**Shape:** A downward-pointing chevron (inverted V) bisected by a single vertical line. The chevron represents a shield or protective barrier. The vertical line bisects it — a spine, a central axis, a signal cutting through protection.

**Construction:** Built on a strict grid. The chevron angle is exactly 60 degrees. The vertical line extends 20% above the chevron's apex and 10% below the base. Stroke weight is uniform — no tapering, no organic qualities. Pure geometry.

**Symbolism:** Protection (the shield form) and surveillance (the vertical line as an antenna or scope). The downward orientation suggests weight, gravity, something pressing down with purpose. For insiders: the bisected shield echoes the half fig leaf without replicating it.

**Variants:**
- Full mark: Chevron + vertical line + "NERV" wordmark in Bebas Neue to the right
- Compact: Chevron + line only (for favicons, app icons, small spaces)
- Wordmark only: "NERV" in Bebas Neue with extended tracking (for text-heavy contexts)

### 5. Voice and Tone

**Voice characteristics:**
- **Declarative.** Statements, not questions. NERV tells; it does not ask.
- **Precise.** No filler. No hedging. Every word is load-bearing.
- **Impersonal.** NERV speaks as an institution, never as "we" or "I."

**Tone: Flat authority.** The emotional register is a straight line. Not cold for the sake of coldness — cold because precision does not require warmth.

**Example copy:**

> Status page header:
> `ALL SYSTEMS NOMINAL. LAST VERIFIED 00:14:22 UTC.`

> Product description (Aeon):
> `Aeon operates 41 autonomous skills across intelligence, development, and system operations. Execution is continuous. Oversight is optional.`

> Error state:
> `PROCESS INTERRUPTED. SKILL: hl-intel. CAUSE: UPSTREAM TIMEOUT. AUTOMATIC RETRY IN 120s.`

> About page:
> `NERV builds systems that operate without interruption. Autonomous agents. Orchestration infrastructure. Gateway services. Each component is designed for unattended execution at scale.`

**Words NERV uses:** nominal, operational, verified, dispatched, processed, active, terminated, queued
**Words NERV avoids:** excited, proud, happy, love, amazing, revolutionary, disruptive, synergy

### 6. EVA Reference Handling

**Strategy: Maximum distance.** The NERV name is the sole reference. Everything else is original. The color palette diverges from EVA's yellow-dominant scheme. The typography avoids any brush/calligraphic weight. The logo is geometrically distinct from the fig leaf. If someone recognizes the reference, it is purely through the name — and even then, "NERV" functions as a credible acronym-style technology brand.

**Risk:** Insiders may feel the reference is too diluted. The brand loses the emotional resonance that the EVA connection provides.

### 7. Strengths and Weaknesses

**Strengths:**
- Maximum credibility with external audiences — reads as a legitimate infrastructure company
- Highly scalable — the institutional voice works at any size
- Clean trademark path — nothing derivative in the visual system
- The voice is distinctive and instantly recognizable
- Strong hierarchy and system thinking — easy to implement consistently

**Weaknesses:**
- Emotionally flat — the existential weight underneath is buried too deep
- May feel generic in the crowded "dark mode enterprise AI" space
- The human element (the EVA DNA of emotional machines) is almost entirely absent
- Team members may not feel connected to the brand — it repels personality
- Risk of feeling like a parody of institutional language without the self-awareness

## APPROACH B: "CONTROLLED TENSION"

**Cold shell, existential core. The institution speaks with mechanical precision, but fractures of something human bleed through the cracks.**

### 1. Name Treatment and Motto

**Name:** NERV (clean, no version number in brand contexts — `_02` reserved for internal/technical designators only)

**Motto:** "All Systems. All Hours."

The motto operates on two levels. Surface reading: 24/7 operational coverage, all systems monitored. Deeper reading: the weight of perpetual responsibility — all systems, always, no rest. It is a promise that doubles as a burden. The ambiguity is the point.

**Sub-entity naming convention:**
- AEON — autonomous operations (no prefix needed — the name carries itself)
- PAPERCLIP — orchestration layer
- OPENCLAW — gateway interface
- Sub-entities are referred to by name alone in most contexts; "a NERV system" is used when institutional framing is needed

### 2. Color Palette

```
PRIMARY
  Signal Orange           #FF6600    (retained — this IS NERV; the existing color has equity)

SECONDARY
  Deep Void               #04040A    (the near-black that is already home)
  Surface Dark            #0D0F14    (card/panel backgrounds)
  Border Gray             #1E2028    (subtle structure lines)

ACCENT
  Caution Yellow          #F6E201    (direct EVA lineage — used for warnings and high-attention states)
  Cold White              #E0E6ED    (text and interface chrome)

NEUTRALS
  Mid Gray                #5A6070    (secondary/disabled text)
  Light Gray              #9BA3B4    (body text)
  Near White              #D4DAE4    (primary text)

STATUS COLORS
  Nominal Green           #34D399    (slightly desaturated — not cheerful, just operational)
  Warning Amber           #FBBF24    (blends with caution yellow family)
  Critical Vermillion     #F43F5E    (rose-red, not pure red — more urgent, less generic)
  Active Blue             #60A5FA    (cool processing indicator)

SPECIAL
  Bleed Orange            #FF660015  (the ghost — used as 8% overlay on dark surfaces to create warmth without brightness)
```

**Rationale:** This palette keeps `#ff6600` as the primary — it already has brand equity in the existing system and among users. The key addition is `#F6E201` (EVA's signature yellow) as a secondary accent. This creates the tension: orange is the operational state, yellow is the warning state. Together they produce a warm-to-hot spectrum against the void-black background. The "Bleed Orange" special token is critical — it is the brand's emotional atmosphere, a barely-visible warmth that exists in the negative space.

### 3. Typography System

**Primary (Headings + UI Labels):** Geist Sans (retained, weight 600-700) — Already in the system. Clean, sharp. Used in ALL CAPS with tight tracking for headers, creating mechanical compression without needing a condensed face.

**Secondary (Body):** Geist Sans (weight 400) — Same family, lighter weight. Continuity matters more than variety.

**Tertiary (Data/Terminal):** Geist Mono (retained) — The backbone of the interface. Terminal output, skill logs, status feeds.

**Display/Emotional (Rare, high-impact):** Noto Serif Display or DM Serif Display — Used ONLY for the motto, for philosophical/existential moments in copy, for the "crack in the shell." This is the brush-weight equivalent — a serif typeface that introduces humanity into an otherwise mechanical system. Appears maybe once per page, if at all. Its rarity is its power.

**The tension rule:** Whenever a serif typeface appears in the NERV system, something human is being expressed. It is the typographic equivalent of a machine pausing to think about what it is doing.

**Hierarchy rules:**
- System headers: Geist Sans 600, 10-11px, ALL CAPS, tracking 0.12em
- Section labels: Geist Sans 500, 12-13px, uppercase, tracking 0.08em
- Body: Geist Sans 400, 13-14px, sentence case
- Data: Geist Mono 400, 10-11px
- Existential/Display: DM Serif Display, 24-48px, sentence case, tracking -0.01em (appears rarely)

### 4. Logo Concept

**Shape:** A geometric shield form — specifically, a hexagonal shape with the bottom vertex extended, creating a downward point. The top edge is flat (stability), the sides angle inward (compression), the bottom comes to a point (direction, weight). Inside the shield, a single thin circle sits slightly off-center — not a target, not a scope, but an eye. Or a zero. Or a period at the end of a sentence.

**Construction:** The outer shield is built from straight lines only. The inner circle is the only curved element — this is the tension. Machine (straight) containing something organic (curved). The circle's off-center placement is deliberate: 40% from center-left, 45% from top. It creates subtle visual unease — something is alive inside the geometry.

**Symbolism:** Protection (shield), containment (the circle within), and the tension between mechanical structure and organic presence. The off-center circle suggests something watching, something that is not quite symmetrical, not quite mechanical. For EVA insiders: the shield echoes the fig leaf's protective function; the inner circle echoes the core/entry plug — something human inside something armored.

**Variants:**
- Full mark: Shield + circle + "NERV" wordmark in Geist Sans 700 to the right
- Compact: Shield + circle only
- Minimal: Circle only (for very small applications — the irreducible element)
- Wordmark: "NERV" in Geist Sans 700, tracking 0.2em

### 5. Voice and Tone

**Voice characteristics:**
- **Precise but aware.** NERV speaks with mechanical clarity but occasionally reveals that it knows the weight of what it is saying.
- **Institutional first person plural.** "We" is used, but it is the "we" of an organization, not a group of friends. Cold "we."
- **Controlled vulnerability.** Not emotional. But not pretending emotions don't exist in the system.

**Tone: Flat with fractures.** The default is institutional calm. But at specific moments — error states, philosophical context, about pages — the tone shifts. A longer sentence appears. A question, maybe. Something that makes you wonder if the system is just reporting status or actually thinking about what the status means.

**Example copy:**

> Status page header:
> `ALL SYSTEMS NOMINAL`
> `Last verified 00:14:22 UTC. Uptime: 847h continuous.`

> Product description (Aeon):
> `Aeon runs 41 skills autonomously — intelligence gathering, code operations, system health, market analysis. It does not sleep. It does not forget.`
> `Whether that is a feature or a condition depends on your perspective.`

> Error state:
> `SKILL INTERRUPTED: hl-intel`
> `Upstream timeout. Retry queued at 120s intervals.`
> `This is routine. Most interruptions are.`

> About page:
> `NERV builds autonomous systems. Agents that operate continuously. Infrastructure that orchestrates without supervision. Gateways that connect intelligence to action.`
> `We named the first agent Aeon because it was supposed to run forever. We are still determining whether that was ambition or oversight.`

> Dashboard footer:
> `All systems. All hours.`

**Words NERV uses:** nominal, continuous, operational, dispatched, verified, queued, autonomous, persistent
**Words NERV sometimes uses:** whether, still, determining, remains, perhaps
**Words NERV avoids:** excited, thrilled, game-changing, innovative, disruptive

### 6. EVA Reference Handling

**Strategy: Distilled DNA.** The EVA reference exists in three deliberate layers:

1. **Name** — the only explicit reference. Functions independently as a technology brand.
2. **Color** — `#F6E201` caution yellow is a direct EVA color, used specifically for warning states. Insiders will recognize it immediately. Outsiders will see a well-chosen warning color.
3. **Emotional architecture** — The "fracture" voice pattern (mechanical text that occasionally becomes self-aware) mirrors EVA's core theme: machines that are more human than they should be. This is never stated, only felt.

Everything else is original. The logo is not a fig leaf. The typography is not Matisse. The color palette is not a reproduction. But the *principles* — mechanical compression, warning-state aesthetics, the tension between institutional control and existential weight — are preserved and translated into an original system.

**For insiders:** The EVA DNA is unmistakable. The yellow warnings, the self-aware error messages, the institution-that-knows-too-much voice. They will recognize the lineage.

**For outsiders:** A sophisticated, slightly unusual technology brand that feels like it has more going on beneath the surface than most.

### 7. Strengths and Weaknesses

**Strengths:**
- The tension IS the brand — no one else in the AI space sounds like this
- Preserves existing color equity (`#ff6600`) while building a richer system around it
- The voice is deeply memorable — the "fracture" moments create emotional connection without sacrificing authority
- EVA DNA is present but never derivative — rewards recognition without requiring it
- Practical to implement — builds on the existing Geist font system
- The serif "tension rule" creates a powerful design system with built-in meaning
- Sub-entities have freedom (their own names) while clearly belonging to the system

**Weaknesses:**
- The voice requires careful execution — the "fracture" moments can feel forced if overdone
- Two warm accent colors (orange + yellow) could create visual confusion without strict usage rules
- The off-center logo element may read as a mistake rather than an intention at small sizes
- The emotional undercurrent may be too subtle for audiences who just want to know what the product does
- More complex to maintain consistently than Approach A's flat authority

## APPROACH C: "SIGNAL NOISE"

**More expressive, leans into warning-state aesthetics. The interface IS the brand — flickering, cascading, alive.**

### 1. Name Treatment and Motto

**Name:** N.E.R.V. (periods between letters — an acronym that never explains itself)

**Designation system:** Each sub-entity gets a classification prefix:
- `SYS::AEON` (autonomous operations)
- `SYS::PAPERCLIP` (orchestration)
- `SYS::OPENCLAW` (gateway)
- `CMD::DASHBOARD` (command interface)

**Motto:** "MAINTAINING PATTERN INTEGRITY"

The motto is a process description, not a promise. It implies that there is a pattern — a system, an order — and that NERV's purpose is to hold it together. The word "maintaining" suggests ongoing effort, tension, the constant work of keeping something from falling apart. It also functions as a loading state, a status message, a thing the system says while it is working.

**Alternative micro-mottos** (used in rotation across the interface):
- `SIGNAL INTEGRITY NOMINAL`
- `PATTERN LOCK ENGAGED`
- `NOISE FLOOR: ACCEPTABLE`
- `CONTINUOUS MONITORING ACTIVE`

### 2. Color Palette

```
PRIMARY
  Warning Orange          #FF6600    (retained — now explicitly framed as a warning color)
  Alert Yellow            #F6E201    (elevated to co-primary — the EVA yellow gets full status)

SECONDARY
  Terminal Black          #020205    (deeper than the current void — almost pure black)
  Panel Dark              #0A0C12    (UI surface color)
  Grid Line               #151820    (structural lines, barely visible)

ACCENT
  Signal Cyan             #00E5FF    (cold counterpoint to the warm primaries — data, links, active states)
  Error Magenta           #FF2D6F    (replaces standard red — more urgent, more designed)

NEUTRALS
  Noise Gray              #3A3E4A    (mid-tone for disabled/inactive)
  Scan Line               #7A8294    (secondary text — named after the visual artifact)
  Phosphor                #C8D0DC    (primary text — the color of old CRT phosphor glow)

STATUS COLORS
  Locked Green            #00FF88    (high-saturation — status colors are BRIGHT in this system)
  Caution Pulse           #FFD600    (animated — this color pulses at 2s intervals when active)
  Critical Flash          #FF2D6F    (the error magenta, with a 0.5s flash animation)
  Processing Sweep        #00E5FF    (the cyan, with a left-to-right sweep animation)
```

**Rationale:** This palette treats colors as signals, not decorations. Orange and yellow share primary status — creating a hot, warning-state atmosphere that is always slightly elevated. The introduction of cyan (`#00E5FF`) provides the cold counterpoint that the other approaches lack — it is the machine's color, the data stream, the thing that is clearly not human. Status colors are deliberately high-saturation and paired with animations. In this system, color is never static. Something is always pulsing, sweeping, flickering. The interface breathes.

### 3. Typography System

**Primary (System/UI):** Space Mono — A monospace typeface with geometric character. Unlike Geist Mono's neutrality, Space Mono has personality — slightly quirky letterforms that suggest a system with its own aesthetic sensibility. Used for ALL interface text including headers.

**Secondary (Data Display):** JetBrains Mono — For dense data readouts, terminal output, log feeds. More readable than Space Mono at small sizes, with programming ligatures for code display.

**Display (Impact):** Space Grotesk — The sans-serif companion to Space Mono. Used for large-scale type: hero sections, splash screens, the N.E.R.V. wordmark. Geometric, slightly quirky, undeniably contemporary.

**Signal Type (Warning States):** The system uses typographic distortion as a semantic tool:
- `letter-spacing: 0.3em` — system is calm, scanning
- `letter-spacing: 0.08em` — system is active, focused
- `letter-spacing: 0em` with `font-weight: 800` — system is in alert state (mechanical compression)
- Text truncation with `...` replaced by `///` — NERV-specific truncation indicator

**Hierarchy rules:**
- System headers: Space Mono 700, 11px, ALL CAPS, tracking 0.2em, orange
- Active headers: Space Mono 700, 11px, ALL CAPS, tracking 0.08em, yellow (compressed = active)
- Body: Space Mono 400, 12-13px, sentence case
- Data: JetBrains Mono 400, 10-11px
- Display: Space Grotesk 700, 36-72px, mixed case, tracking -0.02em

### 4. Logo Concept

**Shape:** Three concentric incomplete circles (arcs) of decreasing radius, centered on a single point. Each arc is approximately 270 degrees — leaving a 90-degree gap. The gaps are staggered at different rotational positions (0, 120, 240 degrees), creating a turbine-like or radar-sweep pattern. At the center: a single solid dot.

**Construction:** The arcs are uniform in stroke weight. The gaps create a sense of rotation and scanning — the mark appears to be in motion even when static. The central dot anchors the composition. When animated (for digital use), the arcs rotate at different speeds, creating a radar/scanning effect.

**Symbolism:** Monitoring (radar), incompleteness (the gaps — nothing is ever fully closed, fully resolved), and convergence (everything pointing toward the center). The three arcs correspond to NERV's three operational domains: intelligence, execution, infrastructure. The central dot is the human operator — or the absence of one.

**Variants:**
- Full animated: Rotating arcs with pulsing center dot (for web/digital)
- Static mark: Fixed arcs + dot (for print/static contexts)
- Full lockup: Mark + "N.E.R.V." in Space Grotesk to the right
- Minimal: Center dot only with a single arc (for favicons)

### 5. Voice and Tone

**Voice characteristics:**
- **Signal-like.** Short bursts. Fragments. The brand communicates like a transmission — essential information only, but with a cadence that creates rhythm.
- **Self-narrating.** NERV describes what it is doing as it does it. The interface is the communication.
- **Oscillating.** The voice alternates between terse status reports and longer, almost poetic observations. Like a radio that picks up two frequencies.

**Tone: Active surveillance.** The brand always feels like it is in the middle of something. Not calm. Not panicked. Actively processing. The resting state is not rest — it is scanning.

**Example copy:**

> Status page:
> `PATTERN INTEGRITY: NOMINAL`
> `41 SKILLS ACTIVE /// 0 FAILURES /// UPTIME 847:14:22`
> `scanning . . .`

> Product description (Aeon):
> `SYS::AEON`
> `Autonomous agent. 41 skills. Continuous execution.`
> `Intel. Code. Markets. Infrastructure. No pause state.`
> `Aeon does not have downtime. Aeon has cycles.`

> Error state:
> `/// SIGNAL INTERRUPTED ///`
> `SKILL: hl-intel | STATUS: UPSTREAM TIMEOUT`
> `RETRY QUEUED: 120s`
> `noise floor elevated. recalibrating.`

> About page:
> `N.E.R.V. builds systems that monitor systems.`
> `Agents that run without asking. Infrastructure that routes without waiting.`
> `Gateways that open before you knock.`
> `The pattern holds. We make sure of it.`

> Loading state (cycling messages):
> `acquiring signal...`
> `pattern lock engaged`
> `noise floor: acceptable`
> `all systems reporting`

**Words NERV uses:** signal, pattern, scan, lock, sweep, nominal, active, cycle, routing, acquiring
**Words NERV avoids:** excited, welcome, please, sorry, unfortunately, we're thrilled

### 6. EVA Reference Handling

**Strategy: Parallel evolution.** The EVA DNA is not hidden but transformed. The warning-state aesthetic is the reference — not any single element, but the entire sensory experience.

1. **Yellow `#F6E201`** is co-primary — an unambiguous EVA color, elevated rather than disguised
2. **Mechanical compression** (tracking changes as semantic signals) directly translates EVA's typographic approach
3. **Breathing interfaces** (pulse animations, sweep effects, cycling status messages) channel the EVA UI philosophy of interfaces that are alive
4. **The rotating logo** echoes EVA's mechanical motion design without referencing any specific EVA visual
5. **`///` as a brand-specific truncation** creates a visual language unique to NERV but inspired by EVA's warning-state framing

**For insiders:** This is unmistakably EVA-inflected. The yellow, the breathing UI, the compressed typography, the relentless operational status reporting. They will feel at home.

**For outsiders:** A highly stylized, distinctive technology brand with a strong visual identity. Unusual, but not referential to anything they would recognize.

### 7. Strengths and Weaknesses

**Strengths:**
- The most visually distinctive of all three approaches — no one else looks like this
- The animated/breathing design system is deeply engaging for the dashboard interface
- Strong internal culture builder — team members will love working within this system
- The EVA DNA is most fully expressed here without being derivative
- The signal/noise metaphor is rich and extensible
- Creates immediate recognition and recall — the brand is unmistakable

**Weaknesses:**
- May read as "too designed" or performative to enterprise/B2B audiences
- The animated elements create accessibility concerns (motion sensitivity, screen readers)
- Two co-primary warm colors (orange + yellow) with cyan accents is a complex palette to manage
- The fragmented voice style can be exhausting over long-form content (docs, blog posts)
- Space Mono is less readable than Geist at small sizes — may create usability friction
- The self-narrating voice risks feeling like an affectation after extended exposure
- Harder to adapt for contexts that require conventional professionalism (investor decks, legal documents)

## RECOMMENDATION

**Approach B: "Controlled Tension" is the strongest direction.** Here is why:

### It solves the actual brand problem

NERV's differentiator is not that it is institutional (Approach A — many AI companies project cold authority). It is not that it has a distinctive visual style (Approach C — style without substance burns out). NERV's differentiator is the tension between what it presents and what it contains: autonomous systems built by someone who thinks about what autonomy means. Approach B is the only direction where this tension is the structural principle of the entire brand.

### It preserves existing equity

Approach B retains `#ff6600`, retains Geist Sans/Mono, and builds on top of the existing UI patterns rather than replacing them. This is not just practical — it is strategically correct. The current system works. It needs depth and intention, not a teardown.

### The voice is unreplicable

The "fracture" voice pattern — mechanical precision that occasionally reveals self-awareness — is the single most defensible brand asset across all three approaches. No other AI company sounds like this. The line *"We named the first agent Aeon because it was supposed to run forever. We are still determining whether that was ambition or oversight"* does something no enterprise brand copy does: it makes you trust the company more because it acknowledges the weight of what it builds.

### The EVA balance is correct

Approach A buries the EVA DNA too deep. Approach C risks wearing it on its sleeve. Approach B distills the principles — warning-state aesthetics, mechanical-organic tension, the institution that knows too much — without reproducing any specific EVA asset. The caution yellow (`#F6E201`) is the key: insiders see it instantly, outsiders see a well-chosen warning color.

### It scales in both directions

Approach B works for a dashboard terminal AND an investor deck AND a GitHub README AND an error page. The voice has a "flat" mode (institutional, precise) and a "fracture" mode (self-aware, weighted), and you choose which to deploy based on context. Approach A only has flat. Approach C only has expressive.

### Implementation path

Approach B requires the least disruptive changes to the existing codebase:
- Keep Geist Sans/Mono as primary type system
- Keep `#ff6600` as primary color
- Add `#F6E201` as accent for warning states
- Add DM Serif Display for rare display/existential moments
- Update copy/voice across existing pages
- Design new logo mark (shield + off-center circle)
- Remove `_02` from brand-facing contexts

### What to borrow from the other approaches

From Approach A, take the formal sub-entity naming when institutional context demands it (NERV/AEON in formal documentation). From Approach C, take the `///` truncation indicator and the cycling status messages for the dashboard loading states — they are too good to leave behind.
